# Tomotto 도전과제 & 배지 시스템 설계

> 버전: v0.1 (2026-05-26)
> 핵심 철학: "시작하게 만들기" — 숫자 노가다보다 행동 유도

---

## 설계 원칙

| ❌ 피할 것 | ✅ 지향할 것 |
|---|---|
| XP 쌓기용 노가다 업적 | 실제 행동 변화를 유도하는 조건 |
| 경쟁심 자극 (빡센 랭크) | 귀여운 압박 + 작은 성취 반복 |
| 복잡한 달성 조건 | 유저가 달성 직후 "아 그거였어!" 하는 직관적 조건 |
| 숫자만 올라가는 보상 | 정체성·세계관을 강화하는 배지 |

## 배지 시스템 개요

### 티어 구조

| 티어 | 조건 난이도 | 배지 외곽선 색 | 수량 |
|---|---|---|---|
| **일반** | 쉽게 달성 가능 | 회색/베이지 | A/C/D 계열 대부분 |
| **희귀** | 꾸준함 또는 다수 반복 | 주황/골드 | B-2,3,5 / C-4,5 / D-5,7 |
| **숨겨짐** | 달성 전 존재 미공개 | 반짝이는 보라/검정 | E 계열 전체 |

### 배지 비주얼 가이드라인

- 모든 배지는 **원형 스티커 메달** 형태
- Tom 계열: 안경 / 체크리스트 / 타이머 / 각진 폼
- Moto 계열: 졸린 눈 / 음표 / 랜덤 큐브 / 흔들리는 표정
- 배틀 계열: 두 토마토 구도 / 하이파이브 / 하트 / 듀오 슬롯머신
- 숨겨진 계열: 어두운 배경 / 반짝임 / 반전 비주얼

## 구현 우선순위

### Phase 1 — localStorage 기반, 즉시 구현 가능

| 업적 | 필요 데이터 |
|---|---|
| A-1 첫 수확 | completedCount ≥ 1 |
| A-2 가챠의 시작 | 가챠 사용 횟수 (기존 gachaCount) |
| A-3 고민 끝! | 가챠 직후 리롤 없이 start 여부 플래그 |
| A-4 이름표 달기 | myNickname 첫 설정 감지 |
| B-6 카테고리 수집가 | categories 배열 길이 |
| B-7 소감가 | 로그의 note 필드 카운트 |
| C-1 일단 돌려! | gachaCount |
| C-3 다양한 수확 | 로그 task 고유값 수 |
| E-3 완전한 하루 | 로그 날짜별 count |
| E-4,5 새벽 감성/마지막 한 판 | 로그 completedAt 시간대 |
| E-6 전설의 토마토 | completedCount ≥ 100 |
| E-8 수집 광 | categories 배열 길이 ≥ 20 |

### Phase 2 — 새 플래그 추가 필요 (소규모 코드)

| 업적 | 추가 필요 |
|---|---|
| A-3, E-7 | `lastGachaRolled` + `startedWithoutReroll` 플래그 |
| B-1,2,3 streak | 날짜별 완료 streak 계산 함수 |
| B-4,5 무결점 | `pauseCount` per session 추적 |
| E-1 몰래 딴짓 | `visibilitychange` 이벤트 카운터 |
| E-2 탈주 | `resetTimer()` 내 remaining ≤ 10 체크 |

### Phase 3 — Supabase 연동 필요

| 업적 | 추가 필요 |
|---|---|
| A-5 초대장 발송 | battles 생성 이벤트 감지 |
| D-1~8 배틀 계열 | battle_players 조회 (파트너 닉, 횟수, 날짜) |
| D-6 한 판 더 | 직전 배틀 completed_at과 현재 배틀 시작 비교 |
| D-7 새벽 농장 | battle_players.proof_uploaded_at 시간대 |

### Phase 4 — 후순위 (MAU 200+ 후 검토)

- 칭호 시스템 (랭크 표시)
- 시즌 한정 배지
- 배지 꾸미기 (슬롯머신 스킨 변경 등)

## 데이터 저장 구조 (제안)

```js
// localStorage: tomotto_achievements
{
  "A-1": { unlocked: true, unlockedAt: 1234567890 },
  "A-2": { unlocked: false },
  "B-1": { unlocked: false, progress: 2 },  // streak 진행도
  ...
}
```

Supabase `user_achievements` 테이블은 Phase 3 이후 추가 (소셜 자랑 기능과 연동 시).
