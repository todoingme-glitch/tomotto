## 🐛 알려진 버그 / 미완성 기능

### 배틀룸 자동 동기화 (v0.1.62 부분 구현)
**증상**: 한 명이 배틀 모달을 닫고 가챠 돌린 뒤 [배틀 열기]를 눌렀을 때, 상대방 화면에도 배틀 모달이 자동으로 떠야 하는데 안 뜨는 경우가 있음.

**현재 구현 방식**: Supabase Broadcast (`room-opened` 이벤트) — 양쪽이 동시에 같은 Realtime 채널에 구독 중일 때만 동작.

**Broadcast가 동작하는 경우 ✅**
- 앱을 켜놓고 다른 앱으로 잠깐 전환했다가 돌아옴 (백그라운드 유지 상태)
- 화면이 꺼졌다가 다시 켜짐

**Broadcast가 안 되는 경우 ❌ (채널이 끊긴 상태)**
- 브라우저 탭을 닫고 URL 다시 입력해서 재접속
- 새로고침 (F5)
- 카카오톡 링크를 다시 탭해서 새 창으로 열었을 때
- 모바일에서 앱을 완전 종료(스와이프 킬) 후 재실행

**개선 방안 (미구현)**
DB 스키마 변경 없이 해결 가능한 방법:
- 배틀룸 열 때 `battle_players` 본인 row에 임시 신호값 UPDATE (예: `viewed_at` 타임스탬프)
- 상대방의 `postgres_changes` 리스너가 이걸 감지 → 모달 오픔
- DB polling 기반이라 새로고침/재접속 후에도 정상 동작
- 구현 난이도: Supabase `battle_players` 테이블에 `room_opened_at timestamptz` 컬럼 추가 필요

## 📋 추후 논의사항 & 방향 결정

### 1. 로그 탭 구조 — 개인/배틀 분리 vs 통합
**결론: 통합**
- 분리하면 배틀 없는 날 한쪽 탭이 비어 보임 + 유저가 탭 두 개 왔다갔다 해야 함
- 통합 뷰 안에서 아이콘·배지로 "혼자 완료" / "배틀 완료" 구분
- 배틀 로그는 상대방 이름이 같이 뜨는 카드로 자연스럽게 구별

### 2. 캘린더 형식
**결론: 현재 월별 스트립 유지** (날짜 원형 가로 나열 + 토마토 꼭지)

### 3. 소셜 탭 추가 콘텐츠
**결론: 아래 3가지 구현 예정**
- **리더보드 (주간/월간)** — 배틀방 멤버 간 완료 횟수 랭킹. 2인 비교만 해도 동기부여 충분
- **활동 상태 표시** — 기능 분리:
  - "내 상태 공개" 토글 → 개인 탭 하단 (설정 역할)
  - 상대 상태 보기 → 소셜 탭 (배틀 카드 내 또는 별도 섹션)
- **공개 배틀방** — 현재 1:1에서 3~5인으로 확장, 모르는 사람과도 집중 세션 공유
- ~~**배틀 초대 히스토리**~~ — 우선순위 낮음, 보류

### 4. 도전과제/배지 탭 위치
**결론: 기록 탭 내 서브탭**
- `기록` 탭 안에 [달력 로그 | 배지] 서브탭 구성
- 별도 탭 분리 시 탭 수가 많아지는 문제
- 배지는 로그를 쌓은 결과물이므로 로그 옆이 맥락상 맞음

## 💰 수익화 방향

**결론: 초기엔 완전 무료 → DAU 확보 후 단계적 적용**

- **현재**: `SHOW_AD_PROMPT = false` 유지 (광고 비활성화 상태가 지금 타이밍에 맞음)
- **1단계 (유저 확보 후)**: 자발적 광고 — 가챠 추가 1회 얻기 버튼 클릭 시만 광고 재생 (리워드 방식). 강제 노출 없음, 거부감 낮음
- **2단계 (서비스 안정 후)**: 프리미엄 구독 검토
  - 유료 기능 후보: 배틀 기록 무제한, 테마/배지 커스터마이징, 상세 통계
  - 집중 앱 유저는 생산성 투자 성향 → 월 몇천원 구독 저항 낮음
- **보류**: 후원/팁 방식 (Buy Me a Coffee 등) — 초기 커뮤니티 기반 서비스에 적합, 필요 시 설정 화면에 추가
- **제외**: 강제 광고 (가챠 4번마다) — 집중 앱 정체성과 충돌, 이탈률 높음

## 📱 네이티브 앱 전환 메모 (Capacitor)

- **타이머 구조**: 현재 코드 이미 `endTime` 기반 timestamp 방식 적용됨. `setInterval`은 화면 표시 갱신 전용(250ms), 실제 시간 계산은 `endTime - Date.now()`. `visibilitychange`에서 포그라운드 복귀 시 자동 보정.
- **남은 갭**: 백그라운드 상태에서 타이머 0 도달 시 `finishTimer()` 미호출 → Capacitor 전환 시 `@capacitor/local-notifications` 플러그인으로 해결 (타이머 시작 시 `endTime`에 로컬 알림 예약)
- **배틀 동기화**: timestamp 방식이 오히려 더 유리 — 양측이 동일한 `endTime` 기준으로 계산하므로 백그라운드 진입 후 복귀해도 자동 동기화
